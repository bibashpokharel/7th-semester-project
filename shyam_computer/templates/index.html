About the Application

Distributed Storage System is a system where user can store and share files securely. Each connected node stores entire files.

Why a Distributed Storage System?

  • Distributed environment
  • Uninterrupted data storage
  • Cost efficiency, secure and reliable
  • Data stored on the system cannot be deleted or tampered with

Apart from the blockchain, Data Share also uses the following technologies:

  • Kademlia Algorithm
    • It is a way for millions of computers to self-organize into a network, communicate with other computers on the network, and share resources (e.g. files, blobs, objects) between computers, all without a central registry or lookup run by a single person or company.
    • It maintain the dynamic hash table.
    • Dynamic hash table includes the entry of K-Bucket which contains IP address, port, and NodeID for peer participants in the system.
    • Resource is located by maping the key
    • Uses XOR routing inorder to locate the resource based on the key provided
  • AES Encryption
    • A file on the IPFS network can be accessed using its unique hash. So, anybody with access to the hash can acccess the file, making it prone to brute force attacks.
    • To eliminate this and giving users enhanced security, Data Share asks the uploader for a key. Using the key, it encrypts the file content first and then uploads it to the IPFS network.
    • A key can be anything which the sender wishes to use as his/her password for the file. The sender shares this key with only those people who he/she trusts for the purpose of receiving the file.
    • To access the file, the recipient needs both the file hash and the key. The key can be shared by the sender using any mode of communication and the file hash gets communicated through the blockchain or the sender.
  • Socket Programming
    • Using Socket Programming, the distributed network is set up such that only the people on the distributed network can view the complete distributed.
